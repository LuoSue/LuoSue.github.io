面向对象及多态

一个Java文件中只能够有一个public类（但是内部类不算）

什么时候一个Java类会有一个默认的无参构造方法？答案是当你没有明确定义一个构造方法的时候，Java类会隐性的定义一个方法体为空的无参构造方法。

创建新对象，代码如下:

Circle circle1 = new Circle();//无参构造方法
Circle circle2 = new Circle(20);//有参构造方法

Java中的对象是通过对象引用变量来进行访问的，这个变量包含了对对象的引用，引用变量如下定义:

ClassName objectVar;

类本质上是一种引用类型，这意味着该类类型的变量均可以引用该类的一个实例。

下面的语句声明myCat的类型是Cat类型

Cat myCat;

myCat能够引用一个Cat对象，下面的语句创建一个对象，并且把它的引用赋给变量myCat:

myCat = new Cat();

下一个句子包括了声明对象引用变量、创建对象并且将对象的引用赋值给这个变量的语句。

ClassName objectRefVar = new ClassName();
Cat myCat = new Cat();

变量myCat中存放的是对Cat对象的一个引用。

其实可以简单的说myCat是一个Cat对象，因为两种说法差异基本不大

在Java中数组也被看作是一个对象，数组用new操作符创建，一个数组变量实际是一个包含数组引用的变量。

基本类型变量与引用类型变量的区别：基本类型变量对应内存所存储的值是基本类型值。对引用类型变量来说，对应内存所存储的值是一个引用，是对象的存储地址。

将一个变量赋给另一个变量：对于基本类型变量而言，就是把一个变量的实际值赋给另一个变量。对于引用类型变量而言，就是将一个对象的引用赋给另一个变量。

当执行完赋值语句之后，c1与c2指向同一个对象，此时c1的对象会被Java运行系统当作垃圾自动回收掉.

不可变对象(immutable object)：创建之后内容不能再改变

不可变类(immutable class)：不可变对象的类

要使一个类成为不可变的，它必须满足下面的要求：

  • 所有数据域都是私有的。
  • 没有修改器方法。
  • 没有一个返回指向可变数据域的引用的访问器方法 。

final关键字：

  • 修饰类：表明这个类不能被继承，类中所有的成员方法均会被隐式地指定为final方法
  • 修饰方法：禁止此方法在子类中被覆盖。类中的private方法会隐式地被指定为final方法
  • 修饰变量：如果变量是基本数据类型的变量，则其数值一旦初始化之后便不能够再被修改；如果变量是引用数据类型的变量，则在对其初始化之后便不能再次让其指向另一个对象

this引用：当引用隐藏数据域以及调用一个重载的构造方法的时候不能够省略。

继承和多态：

如果一个类C1扩展自另一个类C2，那么就把C1称作次类(subclass)，把C2称为超类(superclass)

子类继承父类的所有可以访问的数据域和方法

继承写法：

public class Circle extends GeometricObject

Java中不允许多继承

super关键字指代父类，可以调用父类中的普通方法和构造方法

调用父类的构造方法：

super(); //无参构造方法
super(parameters); //有参构造方法

//如果要调用则必须出现在子类构造方法的第一条语句

构造方法链：构造方法可以调用重载的构造方法或者是父类的构造方法。如果都没有被显式调用，自动调用super()作为构造方法第一条语句。

构造一个类的实例时，将会调用沿着继承链的所有父类的构造方法，构造子类对象时，子类的构造方法会在完成自己的任务之前，先调用父类的构造方法；如果父类又继承自其他类，则会继续调用它自己父类的构造方法，这个过程持续至最后一个构造方法被调用为止。

调用父类的方法：

super.FuntionName(parameters);

(a) 输出的是 A’s no-arg constructor is invoked

(b) B的默认构造方法想要调用A的默认构造方法，可是A中没有默认构造方法

要重写一个方法，需要在子类中使用和父类一样的签名以及一样的返回值类型来对该方法进行定义。
子类从父类中继承方法。有时，子类需要修改父类中定义的方法的实现，这称作方法重写（method overriding)

重载意味着使用同样的名字但是不同的签名来定义多个方法。重写意味着在子类中提供一个对方法的新的实现。

Java中所有的类均继承自java.lang.Object类

多态意味着父类的变量可以指向子类对象。

动态绑定：

方法可以在沿着继承链的多个类中实现。JVM 决定运行时调用哪个方法。

比如说如下代码:

Object o = new GeometricObject();
System.out.println(o.toString());

这里的o会调用哪一个toString()呢？

一个对象会有一个声明类型和实际类型

声明类型：变量被声明时候的类型，此时o的声明类型是Object

实际类型：指被变量引用的对象的实际类，此时o的实际类型是GeometricObject,因为o指向使用new GeometricObject()创建的对象

调用哪一个toString()取决于o的实际类型而定，这称之为动态绑定(dynamic binding)

动态绑定工作机制：

这里说明下：

  • 类的实例包含本身的实例，以及所有直接或间接子类的实例
  • instanceof左边显式声明的类型与右边操作元必须是同种类或存在继承关系，也就是说需要位于同一个继承树，否则会编译错误

equals()

在Object类中定义，测试两个对象是否相等

调用语法：

object1.equals(object2)

protected

经常需要允许子类访问定义在父类中的数据域或方法，但不允许非子类访问这些数据域和方法。可以使用关键字Protected 完成该功能。父类中被保护的数据域或方法可以在它的子类中访问

抽象类和接口

抽象类不能够用于创建对象，抽象类可以包含抽象方法，这些方法会在具体的子类中实现

定义抽象类：

abstract ClassName

抽象方法是非静态的，非抽象类的抽象类子类必须实现抽象方法，抽象的抽象子类不需要也不行

父类是具体的，子类可以是抽象的

接口：只包含常量和抽象方法

Java用如下的语法定义接口：

修饰符 interface 接口名{
    /** 常量声明 */
    /** 方法签名 */
}

与抽象类相似，不能够使用new来创建接口的实例，需要使用一个类来implements一个接口（也可以称作继承）

一个类可以实现多个接口，但是只能继承一个父类

变量 构造方法 方法
抽象类 无限制 子类通过构造方法链调用构造方法，抽象类不能用new操作符实例化 无限制
接口 所有的变量必须是 public static final 没有构造方法，接口不能用new操作符实例化 所有方法必须是公共的抽象实例方法

Java只允许对类的扩展做单一继承，但是接口可以使用多重扩展，例如

public class NewClass extends BaseClass implements Interface1,...,InterfaceN{}

利用关键字extends，接口可以继承其他的接口。这样的接口我们称之为子接口

public interface NewInterface extends Interface1,...,InterfaceN{}

事件处理写法

不是所有对象都可以成为一个动作事件的处理器。要成为一个动作事件的处理器，必须满足两个要求：
    1 ) 该对象必须是EventHandler <T extends Event> 接口的一个示例。接口定义了所有处理器的共同行为。<T extends Event> 表示T 是一个Event 子类型的泛型。
    2)EventHandler 对象handler 必须使用方法source.setOnAction(handler)和事件源对象注册。
    EventHandler <ActionEvent> 接口包含了handle(ActionEvent)方法用于处理动作事
件。你的处理器类必须覆盖这个方法来响应事件。

要实现事件处理：

  1. 定义处理类，处理类需要implements EventHandler <ActionEvent>用于响应事件
  2. 覆盖handle(ActionEvent e)方法，实现事件的响应
  3. 新建并初始化一个处理对象，调用需要处理对象的setOnAction(handler)函数，实现事件处理

代码示例:

import javafx.application.Application;
import javafx.event.ActionEvent;
import javafx.event.EventHandler;
import javafx.geometry.Pos;
import javafx.scene.Scene;
import javafx.scene.control.Button;
import javafx.scene.layout.HBox;
import javafx.stage.Stage;

public class Demo extends Application {
    @Override
    public void start(Stage primaryStage){
        HBox pane = new HBox(10);
        pane.setAlignment(Pos.CENTER);
        Button btOK = new Button("OK");
        Button btCancel = new Button("Cancel");
        OKHandlerClass handler1 = new OKHandlerClass();
        btOK.setOnAction(handler1);
        CancelHandlerClass handler2 = new CancelHandlerClass();
        btCancel.setOnAction(handler2);
        pane.getChildren().addAll(btOK,btCancel);
        Scene scene = new Scene(pane);
        primaryStage.setTitle("HandleEvent");
        primaryStage.setScene(scene);
        primaryStage.show();
    }
}
class OKHandlerClass implements EventHandler<ActionEvent>{
    @Override
    public void handle(ActionEvent event) {
        System.out.println("OK button clicked");
    }
}
class CancelHandlerClass implements EventHandler<ActionEvent>{
    @Override
    public void handle(ActionEvent event) {
        System.out.println("Cancel button clicked");
    }
}

上述函数实现了事件处理的实现

事件是从一个事件源上产生的对象。触发一个事件意味着产生一个事件并委派处理器处理该事件。

产生一个事件并且触发它的组件称之为事件源对象，或者简单称之为源对象或者源组件。

内部类

嵌套类，定义在另一个类范围中的类

大概长这样：

public class A{
    public class B{

    }
}
//B是A的一个内部类

内部类的如下特征：

  • 一个内部类可以引用定义于它所在的外部类中的数据和方法，所以不需要将外部类对象的引用传递给内部类的构造方法
  • 内部类可以使用可见性修饰符所定义，与应用于一个类中成员的可见性规则一样
  • 一个内部类可以被定义为static。一个static的内部类可以使用外部类的名字所访问。一个static的内部类不能够访问外部类中非静态的成员

匿名内部类处理器

匿名内部类：一个没有名字的内部类，一步实现定义一个内部类以及创建一个内部类的实例

匿名内部类的语法如下所示：

new SuperClassName/Interface(){
    //实现或者重写超类或者是接口中的方法

    //其他的方法(如果有)
}

具有以下特征：

  • 一个匿名内部类必须总是从一个父类继承或者是实现一个接口，但是它不能有显式的extends或者implements子句
  • 一个匿名内部类必须实现父类或者接口中的所有抽象方法
  • 一个匿名内部类总是使用它父类的无参构造方法来创建一个实例，如果一个匿名内部类实现一个接口，构造方法是Object()

package Unit09;

或者是:

一个参数的数据类型可以不显式声明，如果只有一个参数，那么可以省略圆括号以及数据类型，效果如下:

e - > {code;}

package JAVAFX;

import javafx.application.Application;
import javafx.geometry.Insets;
import javafx.geometry.Pos;
import javafx.scene.Scene;
import javafx.scene.control.Button;
import javafx.scene.control.Label;
import javafx.scene.control.TextField;
import javafx.scene.layout.FlowPane;
import javafx.scene.layout.GridPane;
import javafx.stage.Stage;

public class Demo extends Application{

    @Override
    public void start(Stage primaryStage){
        GridPane gridPane = new GridPane();
        //设置边距
        gridPane.setPadding(new Insets(10,10,10,10));
        //行列之间距离
        gridPane.setVgap(10);
        gridPane.setHgap(10);
        Label nameLabel = new Label("username:");
        GridPane.setConstraints(nameLabel,0,0);
        TextField nameField = new TextField();
        nameField.setPromptText("username");
        GridPane.setConstraints(nameField,1,0);
        Label passwordLabel = new Label("password:");
        GridPane.setConstraints(passwordLabel, 0, 1);

        TextField passwordField = new TextField();
        passwordField.setPromptText("password");
        GridPane.setConstraints(passwordField, 1, 1);

        Button button = new Button("Log in");
        GridPane.setConstraints(button, 1, 2);

        gridPane.getChildren().addAll(nameLabel,nameField,passwordLabel,passwordField,button);

        Scene scene = new Scene(gridPane , 400, 300);
        gridPane.setAlignment(Pos.CENTER);

        primaryStage.setScene(scene);
        primaryStage.show();
    }

    public static void main(String[] args) {
        Application.launch(args);
    }
}

//BallPane.java
import javafx.animation.KeyFrame;
import javafx.animation.Timeline;
import javafx.beans.property.DoubleProperty;
import javafx.scene.layout.Pane;
import javafx.scene.paint.Color;
import javafx.scene.shape.Circle;
import javafx.util.Duration;

public class BallPane extends Pane {
    public final double radius = 20;
    private double x = radius, y = radius;
    private double dx = 1, dy = 1;
    private Circle circle = new Circle(x, y, radius);
    private Timeline animation;

    public BallPane() {
        circle.setFill(Color.BLUE);
        getChildren().add(circle);
        animation = new Timeline(new KeyFrame(Duration.millis(50),e->moveBall()));
        animation.setCycleCount(Timeline.INDEFINITE);
        animation.play();
    }

    public void play() {
        animation.play();
    }

    public void pause() {
        animation.pause();
    }

    public void increaseSpeed() {
        animation.setRate(animation.getRate() + 0.1);
    }

    public void decreaseSpeed() {
        animation.setRate(animation.getRate() > 0 ? animation.getRate() - 0.1 : 0);
    }

    public DoubleProperty rateProperty(){
        return animation.rateProperty();
    }

    protected void moveBall(){
        if(x<radius||x>getWidth()-radius)
            dx*=-1;
        if(y<radius||y>getHeight()-radius)
            dy*=-1;
        x+=dx;
        y+=dy;
        circle.setCenterX(x);
        circle.setCenterY(y);
    }
}

异常处理写法

直接让一定会抛出错误的代码抛出一个错误

if(Divisor == 0)
    throw new ArithmeticException("Divisor cannot be zero");

throw语句的执行就是抛出一个异常，但一般程序会自己抛出异常，我们只用捕获就好

try{
    Code may throw an exception;
}catch(exceptionType e){
    Code to process the exception;
}

Throwable是所有异常类的基类

可以显式地声明一个方法可能抛出的异常，关键字throws，

格式是public void myMethod() throws Exception1,...,ExceptionN{}

如果方法没有在父类中声明异常，那么就不能在子类中对其进行继承来声明异常

异常处理程序跳转次序

把子类放前边，父类放后边

多个catch块存在时，只会执行一个，当一个方法或者语句抛出异常的时候，跳过其在try-catch块中的之后的所有语句

如果一个异常没有在当前的方法中被捕获，就传给该方法的调用者，重复这个过程直到异常被捕获或者是传给了main方法

  • 异常类型为Exception3：它会被method2中处理异常ex3的catch块捕获，跳过statement5，执行statement6
  • 异常类型为Exception2：退出method2，控制被返回给method1，这个异常会被method1中处理异常ex2的catch块捕获，跳过statement3，然后执行statement4
  • 异常类型为Exception1：退出method1，控制被返回给main方法，这个异常会被main方法中处理异常ex1的catch块捕获，跳过statement1，执行statement2
  • 如果没有被捕获，则程序终止，不执行statement1和statement2

多线程实现方法

多线程实现方法：

Runnable:

一个接口，此接口中只包含了一个run()方法

public interface Runnable {
    public abstract void run();
}

Runnable的作用，实现多线程。我们可以定义一个类A实现Runnable接口；然后，通过new Thread(new A())等方式新建线程。

Thread:

一个类，Thread本身就实现了Runnable接口。它的声明如下：

public class Thread implements Runnable {}

二者的异同点：

  • 相同点：都能够实现多线程
  • 不同点：Thread 是类，而Runnable是接口；Thread本身是实现了Runnable接口的类。我们知道“一个类只能有一个父类，但是却能实现多个接口”，因此Runnable具有更好的扩展性。此外，Runnable还可以用于“资源的共享”。即，多个线程都是基于某一个Runnable对象建立的，它们会共享Runnable对象上的资源。通常，建议通过“Runnable”实现多线程

start()和run()的区别说明：

start() : 它的作用是启动一个新线程，新线程会执行相应的run()方法。start()不能被重复调用。
run() : run()就和普通的成员方法一样，可以被重复调用。单独调用run()的话，会在当前线程中执行run()，而并不会启动新线程

synchronized关键字原理：

相当于加上一个同步锁，当我们调用某个对象的synchronized方法的时候，就相当于获取了该对象的同步锁。

不同的线程对之访问是互斥的，一个对象的同步锁只能够被一个线程获取到。例如，此时有两个线程，线程A和线程B，它们都会访问“对象obj的同步锁”。线程A如果持有同步锁，线程B此时就不能够获取同步锁，它必须等待直到线程A释放了同步锁之后线程B才能运行

join()、wait()、notify()的作用以及用法

wait()：让当前线程处于“等待（阻塞）状态”（直到其他线程调用此对象的notify()方法或者是notifyAll()方法，这样当前线程被唤醒（进入就绪状态））

notify()：唤醒在此对象监视器上等待的单个线程

notifyAll()：唤醒在此对象监视器上等待的所有线程

join() ：让“主线程”等待“子线程”结束之后才能继续运行。

简单的生产者消费者模型：

public class Demo {
    private static Warehouse warehouse = new Warehouse(100);
    public static void main(String[] args) {
        Thread t1 = new Thread(new Consumer(),"Consumer");
        Thread t2 = new Thread(new Producer(),"Producer");
        t1.start();
        t2.start();
    }
    public static class Producer implements Runnable{
        @Override
        public void run() {
            while (true){
                warehouse.produce((int)(Math.random()*10)+1);
                try {
                    Thread.sleep(1000);
                }catch (InterruptedException e){
                    e.printStackTrace();
                }
            }
        }
    }

    public static class Consumer implements Runnable{
        @Override
        public void run() {
            while(true){
                warehouse.consume((int)(Math.random()*10)+1);
                try {
                    Thread.sleep(1000);
                }catch (InterruptedException e){
                    e.printStackTrace();
                }
            }
        }
    }

    public static class Warehouse{
        private int capacity;
        private int size;

        public Warehouse(int capacity){
            this.capacity = capacity;
            this.size = 0;
        }

        public int getSize(){
            return size;
        }

        public synchronized void consume(int amount){
            try {
                int left = amount;
                while (left > 0) {
                    while (size <= 0)
                        wait();
                    int dec = (size<left)?size:left;
                    size-=dec;
                    left-=dec;
                    System.out.printf("%s consume(%3d) <-- left=%3d, dec=%3d, size=%3d\n",
                            Thread.currentThread().getName(), amount, left, dec, size);
                    notifyAll();
                }
            }catch (InterruptedException e){
                e.printStackTrace();
            }
        }

        public synchronized void produce(int amount){
            try{
                int left =amount;
                while(left>0){
                    while(size>=capacity)
                        wait();
                    int inc = (size+left)>capacity?(capacity-size):left;
                    size+=inc;
                    left-=inc;
                    System.out.printf("%s produce(%3d) --> left=%3d, inc=%3d, size=%3d\n",
                            Thread.currentThread().getName(), amount, left, inc, size);
                    notifyAll();
                }
            }
            catch (InterruptedException e){
                e.printStackTrace();
            }
        }
    }
}

import java.net.DatagramPacket;
import java.net.DatagramSocket;
import java.util.Arrays;
import java.util.Date;

public class Server1 {
    public static void main(String[] args) {
// The byte array for sending and receiving datagram packets
        byte[] buf = new byte[1024];

        // Create a packet for receiving data with DatagramPacket(buffer, length)
        DatagramPacket rcvPkt = new DatagramPacket(buf, buf.length);

        // Create a packet for sending data with DatagramPacket
        DatagramPacket sndPkt = new DatagramPacket(buf, buf.length);

        // Create a server socket with DatagramSocket(port)
        try (DatagramSocket socket = new DatagramSocket(8000)) {
            // Print information about server started
            System.out.println("Server started on port 8000, " + new Date());
            while (true) {
                // Initialize buffer for each iteration with Arrays.fill()
                Arrays.fill(buf, (byte) 0);
                rcvPkt.setData(buf);
                // Receive radius from the client socket in a packet
                socket.receive(rcvPkt);

                // Print the client hostname and port number with DatagramPacket.getAddress/getPort
                System.out.println("Received packet from "
                        + rcvPkt.getAddress().getHostAddress()
                        + ":" + rcvPkt.getPort());
                // 123.2.1.4:8000
                // Print the received radius
                var radius = Double.parseDouble(new String(buf).trim());
                System.out.println("Radius: " + radius);

                // Compute area
                double area = radius * radius * Math.PI;
                System.out.println("Area is " + area + '\n');

                // Send area to the client in a packet, with DatagramPacket.setAddress/setPort/setData/send
                sndPkt.setAddress(rcvPkt.getAddress());
                sndPkt.setPort(rcvPkt.getPort());
                buf = Double.valueOf(area).toString().getBytes();
                // TODO 发送的数组可能超过一个Double大小，检查收端是否正确接收
                sndPkt.setData(buf, 0, buf.length);

                socket.send(sndPkt);

                System.out.println("Area sent.");
            } // end while
        } catch (Exception ex) {
            ex.printStackTrace();
        }
    }
}

import java.io.DataInputStream;
import java.net.DatagramPacket;
import java.net.DatagramSocket;
import java.net.ServerSocket;
import java.net.Socket;
import java.util.Arrays;
import java.util.Random;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.locks.Condition;
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

enum Kid {A, B}//定义A，B两个孩子

public class Server {
    RandomInt r = new RandomInt();//随机数
    Lock lock = new ReentrantLock();//互斥锁
    Condition cond = lock.newCondition();//状态变量

    public static void main(String[] args) {
        System.out.println("服务器启动......");
        try {
            ServerSocket ss;
            DatagramSocket ds;
            ss = new ServerSocket(8000);
            ds = new DatagramSocket(10000);
            Server server = new Server();
            ExecutorService executor = Executors.newCachedThreadPool();
            for (int i = 0; i < 10; i++) {//读十次
                Socket s = ss.accept();
                executor.execute(server.new ServerThread(s, Kid.A));
                executor.execute(server.new ServerThread(ds, Kid.B));
                System.out.println("A:char" + "\t" + "A:score" + "\t" + "B:char" + "\t" + "B:score");
                executor.execute(server.new JudgeThread());
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    public class ServerThread implements Runnable {//用于接收TCP以及UDP传输的数据
        private Socket socket = null;
        private DatagramSocket datagramSocket = null;
        Kid kid;//孩子种类
        byte[] buf = new byte[1024];

        public ServerThread(Socket socket, Kid kid) {
            this.socket = socket;
            this.kid = kid;
        }

        public ServerThread(DatagramSocket datagramSocket, Kid kid) {
            this.datagramSocket = datagramSocket;
            this.kid = kid;
        }

        public void ServerWork() {//用于接收TCP和UDP两种数据
            try {
                if (this.kid == Kid.A) {//TCP
                    lock.lock();//上锁
                    var in = new DataInputStream(socket.getInputStream());
                    r.setaKidInt(in.readInt());//读入猜拳结果
                    r.setStatus(kid, true);//设置状态为已出手
                    cond.signalAll();//通知其他线程
                    while (!r.isClean()) {//如果A,B均处于有人出拳
                        cond.await();//等待
                    }
                } else {//UDP
                    lock.lock();//上锁
                    DatagramPacket rcp = new DatagramPacket(buf, buf.length);
                    Arrays.fill(buf, (byte) 0);
                    rcp.setData(buf);
                    datagramSocket.receive(rcp);//读入猜拳结果
                    r.setbKidInt(Integer.parseInt(new String(buf).trim()));
                    r.setStatus(kid, true);//设置状态为已出手
                    cond.signalAll();//通知其他线程
                    while (!r.isClean()) {//如果A,B均处于有人出拳
                        cond.await();//等待
                    }
                }
            } catch (Exception e) {
                e.printStackTrace();
            }
        }

        @Override
        public void run() {
            Random random = new Random(); // 生成随机睡眠时间
            try {
                while (true) {
                    Thread.sleep(random.nextInt(1000));
                    this.ServerWork();
                    lock.unlock();//解锁
                }
            } catch (Exception e) {
                e.printStackTrace();
            }
        }
    }

    public class JudgeThread implements Runnable {
        int aScore = 0;
        int bScore = 0;

        public void Winner() {//判断谁是最后的赢家
            if (this.aScore > this.bScore)
                System.out.println("A is Winner!");
            else if (this.aScore == this.bScore)
                System.out.println("No Winner!");
            else
                System.out.println("B is Winner!");
        }

        public void JudgeWork() {//判断工作
            lock.lock();//上锁
            try {
                while (!r.isReady())//当A，B均没有准备好时
                    cond.await();//等待
                aScore += r.getScore(Kid.A);//计总分
                bScore += r.getScore(Kid.B);
                System.out.print(r.getAnswer(Kid.A) + "\t\t" + r.getScore(Kid.A) + "\t\t" + r.getAnswer(Kid.B) + "\t\t" + r.getScore(Kid.B) + "\n");
                r.ResetKid();//重新设置状态位
                cond.signalAll();//通知其他所有线程
            } catch (Exception e) {
                e.printStackTrace();
            } finally {
                lock.unlock();//解锁
            }
        }

        @Override
        public void run() {
            for (int j = 0; j < 10; j++) {
                this.JudgeWork();
            }
            System.out.println("A's total score: " + aScore);
            System.out.println("B's total score: " + bScore);
            this.Winner();
        }
    }

    class RandomInt {//随机数类，用于生成随机数
        private int aKidInt;
        private int bKidInt;
        boolean aStatus = false;
        boolean bStatus = false;

        public int Compete() {//计算猜拳结果
            if (aKidInt == 0) {
                if (bKidInt == 0)
                    return 1;
                if (bKidInt == 1)
                    return 2;
                if (bKidInt == 2)
                    return 0;
            } else if (aKidInt == 1) {
                if (bKidInt == 0)
                    return 0;
                if (bKidInt == 1)
                    return 1;
                if (bKidInt == 2)
                    return 2;
            } else if (aKidInt == 2) {
                if (bKidInt == 0)
                    return 2;
                if (bKidInt == 1)
                    return 0;
                if (bKidInt == 2)
                    return 1;
            }
            return 0;
        }

        public int getScore(Kid nowKid) {//获得分数
            if (nowKid == Kid.A)
                return Compete();
            else
                return 2 - Compete();
        }

        public void setaKidInt(int aKidInt) {//设置出手
            lock.lock();
            this.aKidInt = aKidInt;
            lock.unlock();
        }

        public void setbKidInt(int bKidInt) {
            lock.lock();
            this.bKidInt = bKidInt;
            lock.unlock();
        }

        public String getAnswer(Kid nowkid) {//返回出了啥
            if (nowkid == Kid.A) {
                if (aKidInt == 0)
                    return "Rock";
                else if (aKidInt == 1)
                    return "Scissors";
                else if (aKidInt == 2)
                    return "Paper";
            } else {
                if (bKidInt == 0)
                    return "Rock";
                else if (bKidInt == 1)
                    return "Scissors";
                else if (bKidInt == 2)
                    return "Paper";
            }
            return "";
        }

        public boolean isReady() {
            return aStatus && bStatus;
        }//是否准备好

        public boolean isClean() {
            return !aStatus && !bStatus;
        }//是否没出拳

        public void ResetKid() {//重设状态位
            aStatus = false;
            bStatus = false;
        }

        public void setStatus(Kid kid, boolean status) {//设置状态位
            if (kid == Kid.A)
                aStatus = status;
            else
                bStatus = status;
        }
    }
